# patFiveCat GitHub 下载与本地演示部署指南

本文档说明如何从 GitHub 下载 `patFiveCat` 项目到本地，并在本机完成演示环境启动。

当前仓库地址：

```bash
git@github.com:PinocchioCat/FiveCat.git
```

如果你使用 HTTPS，也可以使用：

```bash
https://github.com/PinocchioCat/FiveCat.git
```

## 1. 环境准备

建议使用 Windows 环境，和当前项目脚本保持一致。

需要提前安装：

- `Git`
- `Python 3.12`
- `Node.js 20+`
- `npm`
- 可选：`Docker Desktop`

可用下面命令简单检查：

```powershell
git --version
python --version
npm --version
docker --version
```

## 2. 从 GitHub 下载项目

### 方式一：使用 Git 克隆

进入你准备存放项目的目录，例如：

```powershell
cd D:\workspace
```

如果你使用 SSH：

```powershell
git clone git@github.com:PinocchioCat/FiveCat.git patFiveCat
```

如果你使用 HTTPS：

```powershell
git clone https://github.com/PinocchioCat/FiveCat.git patFiveCat
```

进入项目目录：

```powershell
cd .\patFiveCat
```

### 方式二：下载 ZIP

1. 打开 GitHub 仓库页面
2. 点击 `Code`
3. 点击 `Download ZIP`
4. 解压到本地目录
5. 进入解压后的项目目录

注意：

- 如果只是运行项目，ZIP 也可以
- 如果后续还要提交代码，建议使用 `git clone`

## 3. 首次本地准备

项目已经提供一键准备脚本，会自动完成以下事情：

- 生成前后端 `.env`
- 安装后端依赖
- 安装前端依赖

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup-local.ps1
```

如果执行成功，说明本地依赖已经准备完成。

## 4. 启动本地演示

### 推荐方式：一键启动演示

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local.ps1
```

启动成功后，默认访问地址：

- 前端演示地址：`http://127.0.0.1:5174/`
- 后端接口地址：`http://127.0.0.1:8000/`
- 后端 Swagger 文档：`http://127.0.0.1:8000/docs`

说明：

- 当前项目支持 Mock 演示模式
- 即使不启动数据库，也可以先跑通大部分页面和演示流程

## 5. 可选：同时启动数据库和 Redis

如果你希望连同基础设施一起启动，可以执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-local.ps1 -WithInfra
```

该命令会额外拉起：

- PostgreSQL + PostGIS
- Redis

前提是你本机已经安装并启动 `Docker Desktop`。

## 6. 如何停止本地演示

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\stop-local.ps1
```

## 7. 手动启动方式

如果你不想使用脚本，也可以手动启动。

### 7.1 启动后端

```powershell
cd .\backend
python -m pip install -e .
copy .env.example .env
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### 7.2 启动前端

新开一个终端窗口：

```powershell
cd .\frontend
npm install
copy .env.example .env
npm run dev -- --host 127.0.0.1 --port 5174
```

## 8. 演示模式说明

当前项目主要是本地演示型项目：

- 前端：`Vue 3 + Vite + Element Plus`
- 后端：`FastAPI`
- 数据层：当前以 Mock Store 为主

这意味着：

- 首页、登录、订单大厅、社区、个人中心都可以本地演示
- 订单状态流转、支付倒计时、接单和评价逻辑可以直接体验
- 即使数据库未接入完整正式存储，演示流程仍然可以跑起来

## 9. 常见问题

### 9.1 前端打不开

检查端口是否被占用：

```powershell
Get-NetTCPConnection -LocalPort 5174 -ErrorAction SilentlyContinue
```

如果被其他进程占用，先停止冲突进程，再重新运行启动脚本。

### 9.2 后端打不开

检查后端端口：

```powershell
Get-NetTCPConnection -LocalPort 8000 -ErrorAction SilentlyContinue
```

查看日志：

```powershell
Get-Content .\.logs\backend.err.log -Tail 100
Get-Content .\.logs\backend.out.log -Tail 100
```

### 9.3 启动失败提示缺少依赖

重新执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup-local.ps1
```

### 9.4 想启用地图取点

需要在：

```text
frontend/.env
```

里配置：

```text
VITE_AMAP_KEY=你的高德地图 Key
```

## 10. 推荐演示路径

如果你要给别人演示，建议按这个顺序：

1. 打开首页
2. 进入登录页，选择宠物主人或铲屎官身份
3. 进入发单大厅，创建新订单
4. 演示待支付倒计时和订单状态变化
5. 切换到接单大厅，演示接单、待服务、服务中、待确认
6. 返回宠物主人视角，确认完成或发起申诉/退款
